# 60-Second oXRM Demo

I built oXRM for people doing outreach from their own machines.

This could be job applications, CV follow-ups, customer outreach, partnerships,
or founder-led sales.

First, I start the local Docker stack.

Then I open the dashboard. This is synthetic demo data, but the workflow is
real: contacts, companies, leads, tasks, events, and follow-ups.

The important screen is the queue. It tells me who needs attention today.

Now I ask the local MCP tools what is due today.

The agent can read the queue, inspect the relationship history, and draft the
next action.

But it does not send anything automatically. oXRM is not a spam bot. It is a
self-hosted workspace for high-context outreach.

The value is simple: your outreach memory, tasks, and follow-ups stay on
infrastructure you control.
