# v0.2 to v0.3 migration

oXRM v0.3 finishes the public rename and local-first launch shape.

## Repository and package names

- GitHub repository: `otcan/oxrm`
- Product name: oXRM
- Root command: `./oxrm`
- Internal script runner: `scripts/oxrm-internal`
- Instance runner: `scripts/oxrm-instance`
- Workspace packages: `@oxrm/*`
- Docker project prefix: `oxrm-<instance>`
- Default database slug: `oxrm`

The root `crm` wrapper was removed. `./ocrm` remains as a deprecated
compatibility wrapper for one release.

## Compatibility kept intentionally

MCP/API compatibility names remain stable:

- MCP tools such as `crm.search_leads`
- MCP resources such as `crm://queue/today`
- `CRM_API_URL`
- `CRM_MCP_URL`

New automation should prefer `OXRM_API_URL`, `OXRM_MCP_URL`, and `./oxrm`.

## Local instances

Use a blank local workspace for real data:

```bash
./oxrm init my-outreach --template blank
```

Use public-safe synthetic data only for demos:

```bash
./oxrm init job-demo --template job-search
./oxrm init outreach-demo --template outreach
```

Backups now work locally without GitHub credentials. Set `BACKUP_GITHUB_REPO`
and `BACKUP_GITHUB_TOKEN` only when you want backup artifacts pushed to GitHub.
